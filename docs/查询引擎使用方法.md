# HospitalStats.QueryEngine 使用方法

> 最后更新：2026-07-01

独立 Oracle 查询执行引擎，零 ASP.NET 依赖。处理 US7ASCII 中文编码、Oracle 10g ROWNUM 分页、动态 SQL 生成。

**核心特性：完全离线运行。** License 验证、Oracle 查询、SQL 生成——全部在本地完成，不需要外网连接。适合医院内网隔离环境。

---

## 一、获取引擎

引擎通过 `.nupkg` 文件分发，**不需要发布到 nuget.org，不需要外网。**

### 方式 1：本地 NuGet 源（推荐）

将 `.nupkg` 文件放到客户的本地目录，添加为本地源：

```bash
# 将包文件（HospitalStats.QueryEngine.1.0.0.nupkg）放到本地目录
mkdir C:\packages
copy HospitalStats.QueryEngine.1.0.0.nupkg C:\packages\

# 添加本地 NuGet 源
dotnet nuget add source C:\packages --name HospitalStats-Local

# 安装
dotnet add package HospitalStats.QueryEngine
```

### 方式 2：直接复制 DLL

将 `HospitalStats.QueryEngine.dll` 复制到项目中，手动添加引用。

### 方式 3：源码引用（AGPL v3）

```xml
<ProjectReference Include="..\HospitalStats.QueryEngine\HospitalStats.QueryEngine.csproj" />
```

### 发布到 nuget.org（可选）

如果以后想通过 `dotnet add package` 直接安装（仍需 NuGet 客户端有外网），可以发布到 [nuget.org](https://nuget.org)。这是可选的，不是必须的。当前版本 `1.0.0` 包路径：

```
F:\HospitalStats\deploy\nupkgs\HospitalStats.QueryEngine.1.0.0.nupkg
```

## 二、License 激活

### 离线模式（推荐，零服务器依赖，不需要外网）

引擎内置 HMAC-SHA256 离线验证器——License Key 本身包含签名后的授权信息（客户名称、到期日期、功能级别），引擎在本地验证签名和过期时间，**全程不需要联网**。

```csharp
// Program.cs / Startup
using HospitalStats.QueryEngine;

// 一行初始化 —— 纯本地验证，不联网
EngineLicense.InitializeOffline("eyJsaWNlbnNlZFRvIjoiWF...<signature>");

// 注册引擎
builder.Services.AddSingleton<IQueryEngine, QueryEngine>();
```

License Key 是一串 Base64 字符串（`payload.signature` 格式），通过邮件/微信/文件发送给客户，客户复制到代码中即可。

### 在线模式（可选，自定义验证服务器）

如果未来需要联网验证（如按调用次数计费），可以注入自定义回调：

```csharp
EngineLicense.Initialize(async (key) =>
{
    using var http = new HttpClient();
    var resp = await http.PostAsync(
        "https://your-server.com/api/license/validate",
        new StringContent($"{{\"key\":\"{key}\"}}"));
    return resp.IsSuccessStatusCode;
}, licenseKey);
```

### AGPL 模式（源码使用，无需 License）

源码编译引用时不调用 `EngineLicense.Initialize()`，引擎自动跳过验证（静默 no-op）。

## 三、基本使用

### 3.1 DI 注册

```csharp
builder.Services.AddSingleton<IQueryEngine, QueryEngine>();
```

### 3.2 构造查询请求

```csharp
var request = new QueryEngineRequest
{
    // Oracle 连接串（明文，调用方负责解密）
    ConnectionString = "User Id=scott;Password=tiger;Data Source=ORCL",

    // US7ASCII 数据库需指定编码（如 "gbk"）
    CharSetOverride = null,  // 或 "gbk"

    // 主表
    MainTable = new EngineTableDef
    {
        TableName = "OUTP_BILL_ITEMS",
        SchemaName = "HOSPITAL",
        Alias = "A"
    },

    // 输出字段
    Fields = new List<EngineFieldDef>
    {
        new() { ColumnName = "ITEM_CODE",  Alias = "项目编码", TableAlias = "A" },
        new() { ColumnName = "ITEM_NAME",  Alias = "项目名称", TableAlias = "A", DataType = "VARCHAR2" },
        new() { ColumnName = "ITEM_PRICE", Alias = "单价",     TableAlias = "A", DataType = "NUMBER" }
    },

    // 筛选条件
    Filters = new List<EngineFilterDef>
    {
        new()
        {
            Id = 1,
            ColumnName = "ITEM_NAME",
            TableAlias = "A",
            DataType = "VARCHAR2",
            Operator = "LIKE",
            DefaultValue = "%阿莫西林%",
            SortOrder = 0
        }
    },

    // 关联表
    Joins = new List<EngineJoinDef>
    {
        new()
        {
            JoinType = "LEFT",
            JoinTable = new EngineTableDef
            {
                TableName = "DEPT_DICT",
                SchemaName = "HOSPITAL",
                Alias = "B"
            },
            LeftColumnName = "DEPT_CODE",
            LeftTableAlias = "A",
            RightColumnName = "DEPT_CODE",
            RightTableAlias = "B",
            SortOrder = 0
        }
    },

    // 原始 SQL（可选，优先级高于 Fields/Joins/Filters）
    RawSql = null,

    // 排序 & 分组
    SortColumn = "A.ITEM_CODE",
    SortDirection = "ASC",
    GroupByColumn = null,

    // 分页
    Page = 1,
    PageSize = 50,

    // 筛选值（key 对应 FilterDef.Id）
    FilterValues = new Dictionary<string, string>
    {
        ["1"] = "%阿莫西林%"
    },

    // 上下文筛选值（如当前用户科室）
    ContextValues = new Dictionary<string, string>
    {
        ["DeptName"] = "内科",
        ["UserId"] = "42"
    },

    // 运行时选项
    Options = new EngineOptions
    {
        QueryTimeoutSeconds = 120,
        MaxRowCount = 50000
    }
};
```

### 3.3 执行查询

```csharp
// 注入 IQueryEngine
var engine = serviceProvider.GetRequiredService<IQueryEngine>();

// 执行
var result = await engine.ExecuteAsync(request);

// 结果
Console.WriteLine($"Total: {result.Total}");
Console.WriteLine($"Page:  {result.Page}/{Math.Ceiling(result.Total / (double)result.PageSize)}");
Console.WriteLine($"Time:  {result.ElapsedMs}ms");

foreach (var row in result.Rows)
{
    // row is Dictionary<string, object?>
    Console.WriteLine(string.Join(", ", row.Select(kv => $"{kv.Key}={kv.Value}")));
}

// result.Columns: 列名列表（按输出顺序）
// result.Rows: 数据行
// result.Total: 总行数（分页前）
// result.ElapsedMs: 耗时（毫秒）
```

### 3.4 导出 Excel

```csharp
request.Page = 1;
request.PageSize = request.Options.MaxRowCount;  // 导出全部

var bytes = await engine.ExportExcelAsync(request);
File.WriteAllBytes("导出结果.xlsx", bytes);
```

### 3.5 获取筛选选项（下拉框数据）

```csharp
var distinctRequest = new DistinctValuesRequest
{
    ConnectionString = "User Id=scott;Password=tiger;Data Source=ORCL",
    CharSetOverride = "gbk",
    SchemaName = "HOSPITAL",
    TableName = "OUTP_BILL_ITEMS",
    TableAlias = "A",
    ColumnName = "ITEM_NAME",
    DataType = "VARCHAR2"

var options = await engine.GetDistinctValuesAsync(distinctRequest);
// ["阿莫西林", "头孢拉定", "布洛芬", ...]
```

## 四、筛选操作符

| 操作符 | SQL 生成 | FilterValues 示例 |
|--------|---------|-------------------|
| `EQ` | `col = :p` | `"阿莫西林"` |
| `NE` | `col != :p` | `"阿莫西林"` |
| `GT` | `col > :p` | `"100"` |
| `GTE` | `col >= :p` | `"100"` |
| `LT` | `col < :p` | `"2024-01-01"` |
| `LTE` | `col <= :p` | `"2024-01-01"` |
| `LIKE` | `col LIKE :p` | `"%阿莫西林%"` |
| `NOT LIKE` | `col NOT LIKE :p` | `"%测试%"` |
| `IN` | `col IN (:p_0, :p_1, ...)` | `"A001,B002,C003"` |
| `NOT IN` | `col NOT IN (:p_0, ...)` | `"TEST1,TEST2"` |
| `BETWEEN` | `col BETWEEN :p_from AND :p_to` | `"100,500"` |
| `NOT BETWEEN` | `col NOT BETWEEN :p_from AND :p_to` | `"2024-01-01,2024-12-31"` |

**操作符前缀覆盖**：FilterValues 中可以用 `::` 前缀覆盖默认操作符：
```
"NOT LIKE::%测试%"   → 使用 NOT LIKE 替代默认操作符
"GTE::2024-01-01"   → 使用 GTE 替代默认操作符
```

## 五、上下文筛选器

上下文筛选器自动从 `ContextValues` 注入值，用户不可见、不可覆盖：

```csharp
Filters = new List<EngineFilterDef>
{
    new()
    {
        Id = 100,
        ColumnName = "DEPT_NAME",
        Operator = "EQ",
        IsContextFilter = true,
        ContextKey = "DeptName"    // 匹配 ContextValues 的 key
    }
},

ContextValues = new Dictionary<string, string>
{
    ["DeptName"] = "内科",   // 自动注入，用户不可覆盖
    ["UserId"] = "42"
}
```

上下文筛选器无值或为空时自动跳过（不报错）。

## 六、US7ASCII 编码处理

对于 US7ASCII 字符集的 Oracle 数据库，设置 `CharSetOverride`：

```csharp
CharSetOverride = "gbk"  // 或 "gb2312" / "gb18030"
```

引擎自动执行：
1. **Count SQL**：`HexEncodeInlineLiterals` 将 SQL 中的中文字面量转为 hex
2. **Data SQL**：`HexEncodeRawSqlColumns` 对字符串输出列包裹 `RAWTOHEX(UTL_RAW.CAST_TO_RAW(...))`
3. **筛选值**：`EncodeNonAsciiValue` 将中文参数值 hex 编码，确保参数传输不损坏
4. **结果解码**：返回前自动调用 `DecodeHexString` / `ConvertEncoding` 还原中文

### 编码工具方法（可单独调用）

```csharp
using HospitalStats.QueryEngine;

// 判断是否字符串类型
EncodingHelper.IsStringType("VARCHAR2");  // true

// 判断是否含中文
EncodingHelper.ContainsNonAscii("内科");  // true

// hex 解码
EncodingHelper.DecodeHexString("C4DABFC6", "gbk");  // "内科"

// 乱码修复
EncodingHelper.ConvertEncoding(garbleText, "gbk");

// SQL 中文字面量 hex 化
EncodingHelper.HexEncodeInlineLiterals(sql, "gbk");
```

## 七、原始 SQL 模式

配置了 `RawSql` 时，引擎优先使用原始 SQL（而非从 Fields/Joins/Filters 拼装）：

```csharp
var request = new QueryEngineRequest
{
    ConnectionString = "...",
    RawSql = @"
        SELECT a.patient_id, a.name, b.dept_name
        FROM clinic_master a
        LEFT JOIN dept_dict b ON a.dept_code = b.dept_code
        WHERE a.visit_date >= '2024-01-01'
    ",
    // Fields 可为空（引擎从 RawSql 解析列名）
    Fields = new List<EngineFieldDef>(),
    // Filters 仍然生效（注入到 RawSql 的 WHERE 子句中）
    Filters = new List<EngineFilterDef> { ... },
    Page = 1,
    PageSize = 50
};
```

引擎处理流程：
1. **验证**：只允许 SELECT / WITH 查询
2. **注入筛选**：`InjectWhereIntoRawSql` 将用户筛选注入到 GROUP BY / ORDER BY 之前
3. **Hex 编码**（US7ASCII）：`HexEncodeRawSqlColumns` 包裹字符串列
4. **分页**：`ROWNUM` 三层子查询
5. **执行**：Dapper 参数化查询

使用 UNION 的 SQL 也支持，引擎会 `SplitUnionBranches` 后在每个分支分别注入筛选。

## 八、License Key 生成

### 生成密钥

```bash
cd F:/HospitalStats-Enterprise/tools/GenerateLicense
dotnet run -- \
  --licensed-to "XX医院信息科" \
  --expires 2027-07-01 \
  --tier full
```

### Key 格式

```
Base64(JSON_payload).Base64(HMAC-SHA256_signature)
```

其中 payload 内容：
```json
{
  "licensedTo": "XX医院信息科",
  "issuedAt": "2026-07-01T00:00:00Z",
  "expiresAt": "2027-07-01T00:00:00Z",
  "tier": "full",
  "modules": ["query", "export", "diagnose"]
}
```

### 验证逻辑

```csharp
// 手动验证（调试用）
var (isValid, licensee, expiry, tier, error) = BuiltInLicenseValidator.Validate(licenseKey);
Console.WriteLine($"Valid: {isValid}, Licensee: {licensee}, Expires: {expiry}, Tier: {tier}");
if (error != null) Console.WriteLine($"Error: {error}");
```

### 防篡改机制

1. HMAC-SHA256 签名 —— 修改 payload 任意字节都会导致签名不匹配
2. `CryptographicOperations.FixedTimeEquals` —— 时序攻击防护
3. `EngineLicense.GetAssemblyHash()` —— 可选的 DLL 完整性校验

## 九、完整集成示例

### .NET 8 Minimal API

```csharp
using HospitalStats.QueryEngine;

var builder = WebApplication.CreateBuilder(args);

// 1. 激活 License
EngineLicense.InitializeOffline(builder.Configuration["License:Key"]!);

// 2. 注册引擎
builder.Services.AddSingleton<IQueryEngine, QueryEngine>();

var app = builder.Build();

// 3. 使用引擎
app.MapPost("/api/query", async (IQueryEngine engine, QueryEngineRequest request) =>
{
    var result = await engine.ExecuteAsync(request);
    return Results.Ok(result);
});

app.Run();
```

### 控制台应用

```csharp
using HospitalStats.QueryEngine;

// 初始化
EngineLicense.InitializeOffline(args[0]); // license key from command line

var engine = new QueryEngine(LoggerFactory.Create(b => b.AddConsole())
    .CreateLogger<QueryEngine>());

var request = new QueryEngineRequest
{
    ConnectionString = "User Id=scott;Password=tiger;Data Source=ORCL",
    MainTable = new EngineTableDef { TableName = "DEPT_DICT", SchemaName = "HOSPITAL", Alias = "D" },
    Fields = new List<EngineFieldDef>
    {
        new() { ColumnName = "DEPT_NAME", TableAlias = "D" }
    },
    Page = 1,
    PageSize = 10
};

var result = await engine.ExecuteAsync(request);
foreach (var row in result.Rows)
    Console.WriteLine(row["DEPT_NAME"]);
```

## 十、常见问题

### Q: 使用引擎需要外网吗？

**不需要。** 引擎从头设计为完全离线运行：
- License 验证：本地 HMAC-SHA256 签名校验，不联网
- Oracle 查询：直连医院内网 Oracle 数据库
- 无遥测、无追踪、无任何外发网络请求
- `.nupkg` 文件通过邮件/微信/内网共享即可分发，不需要 nuget.org

适合医院物理隔离的内网环境。

### Q: 跟 nuget.org 什么关系？

没关系。nuget.org 是微软的公共 NuGet 包仓库（类似 npm），把包发布到那里后可以通过 `dotnet add package` 直接安装。但这需要外网，且不是必须的。

当前的包就是 `HospitalStats.QueryEngine.1.0.0.nupkg` 文件（36 KB），可以直接复制到客户机器，通过本地源安装。是否发布到 nuget.org 完全可选。

### Q: AGPL 协议对我有什么影响？

如果你用引擎开发的产品也是开源且使用 AGPL 兼容协议 —— 免费使用，无需 License Key。

如果你用引擎开发闭源商业产品 —— 需要购买商业 License。商业 License 包含免责条款、技术支持和优先 Bug 修复。

### Q: License Key 过期后会怎样？

引擎会在 7 天宽限期内继续工作（防止验证服务故障导致生产中断），但会抛出异常提示续费。宽限期从最后一次成功验证起算。

### Q: 可以不调用 EngineLicense.Initialize 吗？

可以。不调用时引擎正常工作（no-op），这适用于 AGPL 源码用户。

### Q: 如何获取商业 License？

联系 is81@qq.com 获取报价和 License Key。
